# frameworks/

ベンダーごとの開発手法・フレームワークの整理ノート。1 ベンダー（または 1 製品）＝ 1 ノートを基本とする。

想定する書き方：

- 全体像（何のためのフレームワークか、対象読者）
- 中核概念（公式が使う用語をそのまま）
- 推奨されるワークフロー
- 他フレームワークとの違い・棲み分け
- `related_sources` に出典の `sources/...md` を必ず並べる

ベンダー横断で「プロンプティング」「エージェント評価」などのテーマを比較したい場合は `frameworks/` ではなく [`../topics/`](../topics/) に書く。
